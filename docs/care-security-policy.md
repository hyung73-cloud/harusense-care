# care security policy

## 분리 원칙

- 가격지도, 지도 DB, 병원/약국 지도 기능과 데이터를 공유하지 않습니다.
- 컬렉션, API, 파일 이름은 `care` prefix를 사용합니다.
- 초기 `clinicId`는 `newsense`입니다.

## 환자 접근

- 환자는 QR 문진 제출만 할 수 있습니다.
- 환자 화면에서는 다른 환자 데이터 조회를 허용하지 않습니다.
- 제출 후 수정/삭제/목록 조회는 의료진만 할 수 있습니다.

## 의료진 접근

- 의료진은 로그인 후 자기 `clinicId` 데이터만 조회할 수 있습니다.
- 의료진 조회, 복사, 수정 이벤트는 `careAuditLogs`에 기록합니다.

## 개인정보와 건강정보

- 이름, 생년월일, 연락처 뒷자리, 체중, 약제, 증상은 민감한 진료 관련 정보로 취급합니다.
- 운영 환경에서는 HTTPS, 서버 인증, DB 권한 규칙, 접근 로그, 백업 정책이 필요합니다.
- 브라우저 저장 방식은 MVP 화면 골격 확인용이며 실제 운영 저장소로 사용하지 않습니다.

## 운영 전 필수 교체

- 데모 로그인 제거
- 서버 기반 인증 적용
- 서버/API 기반 제출 저장 적용
- DB 보안 규칙 적용
- AI API 키 서버 보관
- 감사 로그 서버 저장
- 관리자 계정 발급/회수 절차 마련
